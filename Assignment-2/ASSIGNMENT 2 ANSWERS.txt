Question 1
Program to Create Three Threads
Requirements
Thread A → Calculate sum of first N prime numbers
Thread B → Print message every 2 seconds
Thread C → Print message every 3 seconds
Thread B and C execute in parallel

Program:

#include <stdio.h>
#include <pthread.h>
#include <unistd.h>

int N;

int isPrime(int num)
{
    if(num < 2)
        return 0;

    for(int i = 2; i * i <= num; i++)
    {
        if(num % i == 0)
            return 0;
    }

    return 1;
}

void* threadA(void *arg)
{
    int count = 0;
    int num = 2;
    long long sum = 0;

    while(count < N)
    {
        if(isPrime(num))
        {
            sum += num;
            count++;
        }
        num++;
    }

    printf("Sum of first %d prime numbers = %lld\n", N, sum);

    pthread_exit(NULL);
}

void* threadB(void *arg)
{
    for(int i = 0; i < 50; i++)
    {
        printf("Thread 1 running\n");
        sleep(2);
    }

    pthread_exit(NULL);
}

void* threadC(void *arg)
{
    for(int i = 0; i < 34; i++)
    {
        printf("Thread 2 running\n");
        sleep(3);
    }

    pthread_exit(NULL);
}

int main()
{
    pthread_t t1, t2, t3;

    printf("Enter N: ");
    scanf("%d", &N);

    pthread_create(&t1,NULL,threadA,NULL);
    pthread_create(&t2,NULL,threadB,NULL);
    pthread_create(&t3,NULL,threadC,NULL);

    pthread_join(t1,NULL);
    pthread_join(t2,NULL);
    pthread_join(t3,NULL);

    return 0;
}

Output:

Sum of first 10 prime numbers = 129

Thread 1 running
Thread 2 running
Thread 1 running
Thread 2 running
Thread 1 running
...

Question 2
Signal Handling Implementation
Objective

Prevent program termination when CTRL+C (SIGINT) is pressed.

Program:

#include <stdio.h>
#include <signal.h>
#include <unistd.h>

void signalHandler(int sig)
{
    printf("\nSIGINT Received. Termination prevented.\n");
}

int main()
{
    signal(SIGINT, signalHandler);

    while(1)
    {
        printf("Program Running...\n");
        sleep(2);
    }

    return 0;
}

Locking Mechanisms
Mutex

Mutual Exclusion Lock.

Purpose:

Only one thread accesses shared resource at a time

Example:

pthread_mutex_lock();
pthread_mutex_unlock();

Advantages:

Prevents race conditions
Easy implementation
Spinlock

A thread continuously checks lock availability.

Example:

while(lock_busy)
{
    spin;
}

Advantages:

Faster for very short waits

Disadvantages:

CPU intensive
Difference Between Mutex and Spinlock
Feature	Mutex	Spinlock
Waiting	Sleeps	Busy Wait
CPU Usage	Low	High
Context Switch	Yes	No
Best For	Long Waits	Short Waits
